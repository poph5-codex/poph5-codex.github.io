# 难度曲线计算器说明

## 这是什么

这是一个本地网页工具，用来把原本只能在 Excel 里阅读和手动试算的难度曲线模型，整理成一个可以交互调参、实时观察结果的可视化工具。

当前版本以参考 Excel 为基础，保留了其中最稳定、最适合抽象成通用模型的部分：

1. 基础增长曲线
2. 50 关周期修正
3. 特殊关修正
4. buff / 连胜修正
5. 最终离散化输出
6. 滚动体验观察指标

## 当前输入来源

- 参考资料：`design/难度曲线11.11最终.xlsx`
- 程序读取副本：`data/reference_model_readonly.xlsx`
- 提取后的种子数据：`data/model_seed.json`
- 我方项目默认调试模板：`data/project_seed_default.json`

## 当前工具的定位

当前版本是“模型调试器”，不是“旧 Excel 的逐格复刻版”。

它的重点是：

1. 把关键参数暴露出来
2. 让策划看到每个参数如何影响曲线
3. 方便以后替换成我方项目自己的真实数据

## 模型主结构

### 1. 基础增长层

用对数公式生成整体难度上升趋势，控制“关卡越往后越难”的主节奏。

### 2. 周期起伏层

用一个 50 关周期模板去修正基础增长结果，制造每个阶段内部的波峰和缓冲段。

### 3. 特殊关修正层

对引导关、金币关、节点关、尾号限制关等特殊关型做额外处理。

### 4. buff / 连胜层

把裸难度换算成玩家真实体感难度，方便模拟不同项目里的 buff 生态。

### 5. 离散化层

把连续难度转换成策划实际使用的离散档位。

### 6. 观察指标层

输出单关结果之外，还会输出：

1. 近 20 关平均难度
2. 近 50 关平均难度
3. 近 100 关平均难度
4. buff 体感指标
5. 前期人工覆盖情况

## 当前页面里主要能调什么

1. 基础增长参数
2. 周期长度与周期强度
3. 周期模板 10 段分布
4. 引导关 / 金币关 / 特殊关修正
5. buff 占比与 buff 难度修正系数
6. 前 50 关人工微调
7. 离散化阈值

## 替换成我方项目数据的建议顺序

1. 先确认我方项目自己的难度目标口径
2. 准备我方关卡基础信息
3. 准备我方线上表现数据
4. 准备我方 buff / 道具 / 复活数据
5. 替换 `data/` 中的结构化输入
6. 再根据实际体验调页面默认参数

## 当前版本限制

1. 还没有做任意 Excel 上传解析
2. 还没有方案版本管理
3. 还没有把旧表里所有辅助标签规则完整搬进来

但当前版本已经足够用于：

1. 新项目难度曲线方案试算
2. 参数敏感性观察
3. 周期与 buff 影响分析
4. 与参考模型做对照
